Select문 

가장 많이 쓰이는 DML(데이터 조작 언어) 질의어

SELECT [DISTINCT] {*, column [Alias], ….}
FROM 데이블명
[WHERE condition]
[ORDER BY {column, expression} [ASC|DESC]] ;

SELECT : 원하는 컬럼 선택
*	: 테이블의 모든 column 출력
alias	: 해당 column에 대한 다른 이름 부여
DISTINCT: 중복 행 제거 옵션
FROM  : 원하는 데이터가 저장된 테이블 명을 기술
WHERE : 조회되는 행을 제한하기 위한 조건 추가
condition : column, 표현식, 상수 및 비교연산자
ORDER BY:정렬을 위한 옵션 (ASC :오름차순이며 default, DECS 내림차순)


SQL 문장 작성법
-SQL 문장은 대소문자를 구별X
-SQL 문장은 한 줄 또는 여러 줄에 입력 가능
-절은 보통 읽고 편집하기 쉽게 줄을 나누도록 권장
-탭과 줄 넣기(들여쓰기)는 코드를 보다 읽기 쉽게 하기 위해 권장
-일반적으로 키워드는 대문자로 입력
-명령어의 끝은 ;
-주석처리는 -- 

SQL 문장 실행
-SELECT * FROM TAB; -- 소유하고 있는 table을 전부 보여줌
SELECT * FROM 테이블명; -- 테이블 데이터 전체 보기
DESC 테이블명;  -- 테이블의 구조보기
SELECT * FROM 테이블명; -- 테이블의 전체 내용
SELECT 컬럼명 FROM 테이블명; -- 테이블의 선택한 컬럼 내용
SELECT 컬럼명 AS "별명", 컬럼명 AS "별명", 컬럼명 AS "별명" FROM 테이블명; -- 테이블의 선택한 컬럼을 별명으로 출력
SELECT 컬럼명 별명, 컬럼명 별명,컬럼명 별명 FROM 테이블명;-- 테이블의 선택한 컬럼을 별명으로 출력


WHERE 조건과 비교연산자
같다(=), 크거나같다(>=) 작거나같다(<=) 다르다(<> ^= !=)
SELECT 컬럼명 테이블명 WHERE 특정 컬럼명 = 비교할 값;
SELECT 컬럼명 FROM 테이블명 WHERE 비교할 컬럼명 <> 비교할 값;
SELECT 컬럼명 FROM 테이블명 WHERE 비교할 컬럼명  != 비교할 값;
SELECT 컬럼명 FROM 테이블명 WHERE 비교할 컬럼명  ^= 비교할 값;
SELECT * FROM 테이블명 WHERE 특정컬럼명=특정값;
SELECT 컬럼명  FROM 테이블명 WHERE 특정컬럼명=’특정값’;
­	SQL에서 문자열이나 날짜는 반드시 작은따옴표안에 표시.
­	SQL문에 사용되는 키워드인 SELECT나 FROM이나 WHERE 등은 대소문자를 구별하지 않지만 테이블 내에       	
        저장된 데이터 값은 대소문자를 구분 
SELECT * FROM 테이블명 WHERE 비교할 컬럼명>=비교할 값1 AND 비교할 컬럼명<비교할 값2;  --비교할 값1,2 사잇값 출력
SELECT * FROM 테이블명 WHERE 날짜컬럼 <= ‘RR/MM/DD’;
­	테이블의 날짜컬럼에서 RR년MM월DD일 이전의  모든 정보를 출력
SELECT  컬럼명, 숫자컬럼, 숫자컬럼에 *,+특정값  "계산한 컬럼명" FROM 테이블명 WHERE 계산한 컬럼명>비교할 값 ORDER BY 숫자컬럼 —error

논리연산자
SELECT * FROM 테이블명 WHERE 특정컬럼명=특정값 AND 특정컬럼명=특정값’;
SELECT * FROM 테이블명 WHERE 특정컬럼명=특정값 OR 특정컬럼명=특정값’;
SELECT * FROM 테이블명 WHERE NOT 특정컬럼명=특정값; 
SELECT * FROM 테이블명 WHERE 특정컬럼명>=특정값1 AND 특정컬럼명<=특정값2;
­	특정값1이상특정값2이하인 특정컬럼명 정보 출력

산술표현식
SELECT 컬럼명, 숫자컬럼명+특정값 AS “” FROM 테이블명;
­	테이블의 숫자컬럼을 특정값 증가 시키기 위해 덧셈연산자를 사용하여 숫자컬럼명+특정값을 출력. 계산한컬럼은 테이블의 새로운 열이 아니고 단지 디스플레이를 위한 것.
SELECT 컬럼명,NULL포함숫자컬럼명, 숫자컬럼명*특정값 1, 숫자컬럼명*특정값+NULL포함숫자컬럼명  계산한컬럼명2 FROM 테이블명;
­	null값의 처리는 행이 특정열에 대한 데이터값이 없다면 그 값은 null
        null값은 이용할 수 없거나 지정되지 않았거나 알 수 없거나 적용할 수 없는 값. 
        Null은 1이나 공백과는 다르고 0은 숫자이며 공백은 문자로 봄
        어떠한 datatype도 null값을 포함할 수 있으나 주키는 null값이 허용X 
­	null값을 포함한 산술표현식 결과는 NULL
NVL함수 를 이용
SELECT 컬럼명,숫자컬럼명,NULL포함숫자컬럼명,숫자컬럼명+NULL포함숫자컬럼명 FROM 테이블명;
SELECT 컬럼명,숫자컬럼명,NULL포함숫자컬럼명, 숫자컬럼명*특정값, 숫자컬럼명*특정값+NVsL(NULL포함숫자컬럼명,0) FROM 테이블명; 
­	NVL(expr1, expr2)  expr1은 null값을 포함하고 있는 column이나 표현식                       
expr2는 null변환을 위한 목표값, expr1과 expr2는 같은 타입이여야 함
­	NVL(hiredate , ‘99/01/01’) DATA형에서의 예
­	NVL(job, ‘업무없음’)      char형에서의 예

연결연산자 (||)
열이나 문자를 다른 열에 연결하는 연산
SELECT 컬럼명1 || ‘ .... ‘ || 컬럼명2 as “별칭” FROM 테이블
-별칭컬럼에 
컬럼명1내용 .... 컬럼2내용
 으로 추출됨
­	컬럼명에 별칭은 프로그램 영역에서 중요한 의미
        대다수 프로그램은 별칭으로 간편하게 사용하는 경우가 많음

중복제거 DISTINCT
DISTINCT를 사용하여 나타나는 결과는 기본적으로 오름차순 정렬
DISTINCT 필드; 필드 중복제거
SELECT DISTINCT 컬럼명 FROM 테이블명;
SELECT DISTINCT 컬럼명1, 컬럼명2 FROM 테이블명;
­	테이블에서 컬럼명1별로 컬럼명2를 한번씩

BETWEEN a AND b : a부터 b (a,b값 포함. a가 작은 값) 
NOT BETWEEN a AND b : a와 b 사이에 있지 않음(a,b값 포함되지 않음)

IN(list)  : list의 값 중 어느 하나와 일치
NOT IN (list)  : list의 값과 일치X

LIKE pattern  : 검색하고자 하는 값을 정확히 모를 경우 pattern과 일치하는지를 검색NOT LIKE pattern  : 문자형태가 pattern과 일치X
           pattern에는 다음의 두가지 와일드카드가 사용
­	% : 문자가 없거나, 하나 이상의 문자가 어떤 값이 와도 상관X
­	_ : 하나의 문자가 어떤 값이 와도 상관X
EX)이름, 상사의사번, 급여 (이름이 M으로 시작)
­	SELECT ename, mgr, sal FROM emp WHERE ename LIKE 'M%';
­	SELECT ename, mgr, sal FROM emp WHERE ename LIKE '%N%'; -- %0글자이상
­	SELECT ename, mgr, sal FROM emp WHERE ename LIKE '_A%'; -- _한글자
­	SELECT ename from emp where ename like ‘K%’; -- 이름이 K로 시작하는 이름
­	SELECT ename from emp WHERE ename LIKE ‘%S’; -- 이름이 S로 끝나는 이름
­	SELECT ename FORM emp WHERE ename LIKE ‘%\%%’ ESCAPE ‘\’; --이름에 %가 들어간 이름


IS NULL : null값을 가짐
NOT 필드 IS NULL (필드 IS NOT NULL) : null값을 갖음(갖지 않음)

IN ()
SELECT 컬럼명 FROM 테이블명 WHERE 컬럼명 IN (특정값); 
컬럼에서 특정값만 출력
